## 🚀 Inicio Rápido

### 1. Instalar Dependencias

```bash
# En Windows PowerShell
cd starter-kit/starter-kit
.venv\Scripts\activate.ps1

# Instalar dependencias de desarrollo
pip install -r requirements-dev.txt
```

### 2. Ejecutar Pruebas (Elige Una Opción)

#### Opción A: Comandos Simples
```bash
# Todas las pruebas con cobertura
pytest tests/ -v --cov=app --cov-report=html

# Solo pruebas unitarias
pytest tests/unit/ -v

# Solo pruebas de seguridad
pytest tests/security/ -v
```

#### Opción B: Usar Makefile (macOS/Linux)
```bash
make help           # Ver todos los comandos
make test           # Ejecutar todas las pruebas
make coverage       # Generar reporte de cobertura
make sec-all        # Ejecutar todos los escaneos de seguridad
```

#### Opción C: Usar el Script Python
```bash
# Todas las pruebas
python run_tests.py --all

# Con cobertura
python run_tests.py --coverage

# Suite completa (todo)
python run_tests.py --full
```

### 3. Ver Reportes

```bash
# Windows
Start-Process htmlcov\index.html

# macOS
open htmlcov/index.html

# Linux
xdg-open htmlcov/index.html
```

## 🔍 Estructura de Pruebas

### Unit Tests (Pruebas Unitarias)

**Ubicación**: `tests/unit/`

**Incluye**:
- Pruebas de hashing de contraseñas
- Validación de usuarios
- Flujo de autenticación
- Cambio de contraseña

```bash
pytest tests/unit/ -v
```

### Integration Tests (Pruebas de Integración)

**Ubicación**: `tests/integration/`

**Incluye**:
- Ciclo de vida completo del usuario
- Aislamiento multi-tenant
- Manejo de errores
- Persistencia de datos

```bash
pytest tests/integration/ -v
```

### Security Tests (Pruebas de Seguridad)

**Ubicación**: `tests/security/`

**Incluye**:
- Prevención de SQL Injection
- Bypass de autorización
- Seguridad de contraseñas
- Protección CSRF
- Session Security
- Rate Limiting

```bash
pytest tests/security/ -v
```

## 📈 Reportes de Cobertura

### HTML Report (Recomendado)

```bash
pytest tests/ --cov=app --cov-report=html
# Abre: htmlcov/index.html
```

Muestra:
- Porcentaje de cobertura por archivo
- Líneas ejecutadas vs no ejecutadas
- Ramas de código cubierta

### Terminal Report

```bash
pytest tests/ --cov=app --cov-report=term-missing
```

Muestra:
- Resumen de cobertura por archivo
- Números de línea no cubierta

### XML Report (para CI/CD)

```bash
pytest tests/ --cov=app --cov-report=xml
```

Usado por:
- SonarQube
- Codecov
- GitHub Actions

## 🔐 Análisis de Seguridad

### Bandit (Escaneo Estático)

Busca:
- Inyecciones SQL
- Uso de funciones inseguras
- Hardcoded passwords
- Problemas de deserialización

```bash
bandit -r app/ -v
bandit -r app/ -ll          # Solo critical
bandit -r app/ -f json -o bandit-report.json
```

### Safety (Verificación de Dependencias)

Busca:
- CVE vulnerabilities
- Dependencias inseguras

```bash
safety check
safety check --critical
safety check --json > safety-report.json
```

### Pylint (Calidad de Código)

Verifica:
- Errores de sintaxis
- Problemas de estilo
- Problemas de diseño

```bash
pylint app/
pylint app/ -d R,W     # Solo errores y críticos
```

## 🎯 SonarQube Integration

### Configuración

1. **Archivo**: `sonar-project.properties`
   - projectKey: mesapass-backend
   - sources: app
   - tests: tests

2. **Cobertura**: Requiere `coverage.xml`

3. **Token**: Necesario para enviar resultados

### Ejecutar Análisis

```bash
# Paso 1: Generar reporte de cobertura
pytest tests/ --cov=app --cov-report=xml

# Paso 2: Ejecutar sonar-scanner
sonar-scanner \
  -Dsonar.projectKey=mesapass-backend \
  -Dsonar.host.url=http://localhost:9000 \
  -Dsonar.login=YOUR_TOKEN

# O simplemente:
sonar-scanner
```

### Métricas que Rastrea

| Métrica | Descripción |
|---------|-------------|
| Code Smells | Problemas de mantenibilidad |
| Bugs | Posibles errores |
| Vulnerabilities | Problemas de seguridad |
| Coverage | % de código cubierto |
| Duplications | Código duplicado |
| Technical Debt | Tiempo estimado para arreglar |

## 🚨 Troubleshooting

### Error: "No module named pytest"

```bash
pip install pytest pytest-cov
```

### Error: "Database locked"

```bash
rm test.db
pytest tests/ -v
```

### Error: "CORS error in tests"

Las pruebas usan cliente de prueba que omite CORS.

### Sonar connection refused

```bash
# Verificar que SonarQube está corriendo
# Default: http://localhost:9000
```

## 📞 Soporte

### Preguntas Frecuentes

**P: ¿Cómo agrego nuevas pruebas?**
A: Crea archivo en `tests/unit/`, `tests/integration/`, o `tests/security/` con prefix `test_`.

**P: ¿Cómo aumento la cobertura?**
A: Usa `pytest tests/ --cov=app --cov-report=term-missing` para ver líneas no cubiertas.

**P: ¿Puedo ejecutar pruebas sin instalar dependencias?**
A: No, necesitas las herramientas de testing.

**P: ¿Qué pasa si una prueba falla?**
A: Lee el error, usa `pytest -s` para ver prints, y `pytest -v` para más detalles.
